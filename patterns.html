Patterns

Common ways cascades accelerate

Patterns let you recognize a cascade early. If you can name the shape, you can intervene before it becomes a rescue mission.

How to read this

Each pattern includes: what it is, why it amplifies failure, and a practical way to interrupt it. Scenarios below show the patterns in motion.

The atlas

Retry Storm

What: Clients retry when things slow down. Amplifier: retries add load, increasing latency, triggering more retries.

Interrupt: cap retries, use exponential backoff + jitter, and track retry rate separately.

Scenario: Retry Storm → Outage

Alert Fatigue

What: Alerts fire too often or too noisily. Amplifier: response slows because the signal isn’t trusted.

Interrupt: reduce noisy alerts, define severity, and practice response drills.

Scenario: “It’s probably fine” → Too late

Hidden Coupling

What: Parts depend on each other more than people realize. Amplifier: one failure spreads impact across boundaries.

Interrupt: isolation, graceful degradation, and clear boundaries with fallback behavior.

Scenario: Coupling Cascade

Queue Collapse

What: Work queues build up under load. Amplifier: backlog increases latency/resources, causing timeouts and failures.

Interrupt: backpressure, drop/limit work early, keep queues bounded.

Workaround Debt

What: Quick fixes accumulate. Amplifier: each workaround changes system behavior and hides underlying fragility.

Interrupt: track workarounds, set expiry dates, fix root causes intentionally.

No Slack

What: The system runs near capacity. Amplifier: small spikes become catastrophic because there’s no margin to absorb variation.

Interrupt: add margin, simplify hot paths, reduce load, or redesign for elasticity.
